Option 01

The solution

Agent Harbor: BYO subscription, unified control

Agent Harbor sits on top of the subscriptions your team already pays for. It doesn't sell you tokens — it orchestrates the ones you have.

How it works

Developers plug in their existing AI subscriptions — company plans or personal. Agent Harbor uses their inference while adding long-horizon execution, advanced sandboxing, and fleet management on top. If an agent maxes out one plan mid-task, Agent Harbor can hot-swap to the next available subscription and continue without losing context.

Use case

Long-running autonomous bug detection and remediation

A real workflow showing how Agent Harbor's capabilities work together for a multi-day, multi-agent engineering task.

1. Trigger from Slack or Telegram

Initiate

An admin sends a message to the team's Agent Harbor bot: "Check the v2.4 release for bugs on macOS and Windows." Agent Harbor spawns a computer-use agent with full browser and OS interaction capabilities in a sandboxed environment.

harbor spawn --mode=computer-use --target=v2.4
→ sandbox "harbor-2847" created (CoW, 14ms boot)

2. Continuous exploratory testing

Running

The explorer agent runs for days, systematically testing edge cases in the release. The anti-halting harness detects stuck states, recovers context, and keeps the agent moving forward. Findings are filed as structured reports.

Agent uptime: 72h 14m
Stall recoveries: 3
Bugs filed: 7
→ Anti-halting harness active

3. Implementer agent picks up fixes

Delegate

Bug reports are delegated to a separate implementer agent with full codebase context. It begins working on fixes autonomously, proposing PRs with patches verified against the isolated sandbox environment.

Leader: explorer-agent → Follower: impl-agent-01
→ Full codebase context passed via leader-follower orchestration

4. Multi-OS validation

Verify

Each fix is tested simultaneously on Windows, macOS, and Linux using Agent Harbor's copy-on-write filesystem. Agents can even run on the same machine, sharing the same ports without conflicts — each in its own isolated copy-on-write sandbox.

Sandboxes: 3 (win / mac / linux)
Disk overhead: <50 MB (CoW)
Port conflicts: 0
→ All running on one host machine

5. Credential hot-swap keeps work going

Resilient

When the long-running explorer agent maxes out its Claude subscription mid-task, Credential Manager hot-swaps to the user's next available plan with no interruption.

claude-max-plan: exhausted
hot-swap → codex-team-plan
→ Context preserved, task continued seamlessly
